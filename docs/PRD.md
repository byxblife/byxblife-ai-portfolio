# 產品需求文件 (PRD)

## 1. 產品願景 (Vision)
建立一個專業且具個人特色的 Personal AI Portfolio 網站，透過**類似 Apple 官網的極簡質感設計與流暢的高品質動畫**，向潛在雇主與技術合作夥伴展示個人的 AI 開發能力、專案成果以及專業背景。

## 2. 使用者人物誌 (Target Persona)
### 主要目標客群：招聘經理 (Hiring Manager) / 技術面試官
- **特徵**：時間寶貴，注重候選人的實際產出與技術深度，同時欣賞高品質的設計細節。
- **痛點**：瀏覽大量履歷令人疲乏，希望能快速找到候選人的亮點與實作能力證明。
- **需求**：需要一個能快速了解候選人「是誰」以及「做過什麼」的入口，並被高品質的呈現方式所吸引。

### 次要目標客群：潛在合作夥伴 / 一般訪客
- **特徵**：對 AI 技術感興趣，尋找合作機會或靈感。
- **需求**：清晰的作品展示、令人印象深刻的視覺體驗與簡單的聯繫方式。

## 3. 功能需求列表 (Feature List) - MoSCoW 排序

### Must Have (必須有)
- **首頁 (Landing Page)**：作為網站的主要入口，需呈現高質感的視覺衝擊。
- **極簡質感設計 (Apple-like Minimalist Design)**：採用大量留白、精選字體與高解析度素材，營造尊榮且專業的氛圍。
- **高品質動畫 (High-Quality Animations)**：包含平滑的轉場、滾動視差 (Parallax) 或微互動 (Micro-interactions)，提升使用者體驗但不可喧賓奪主。
- **自我介紹區塊 (About Me Introduction)**：包含個人簡介、核心技能與職涯背景的摘要。
- **作品集連結列表 (Portfolio Links)**：展示精選專案的標題與連結，點擊後可跳轉至專案詳情或外部演示連結。

### Should Have (應該有)
- **響應式設計 (Responsive Design)**：確保在桌面電腦、平板與手機上皆能良好顯示，並保持設計語言的一致性。
- **社群媒體連結**：GitHub、LinkedIn 等專業社群的連結圖示，建議整合於頁尾或導覽列。

### Could Have (可以有)
- **詳細作品頁面**：每個作品有獨立的詳細說明頁面，包含技術堆疊與開發心得。
- **聯絡表單**：讓訪客能直接透過網站寄送訊息。

### Won't Have (本次不會有)
- **後台管理系統 (CMS)**：初期內容直接寫在程式碼中 (Hardcoded) 或透過簡單的設定檔管理，不開發複雜的後台。
- **使用者登入系統**：本站為公開展示用途，不需會員功能。

## 4. 驗收標準 (Acceptance Criteria)
1.  **首頁載入測試**：使用者進入網站首頁時，應在 3 秒內完成載入，並立即看到自我介紹內容（包含初始進場動畫）。
2.  **視覺風格驗收**：整體介面應符合「極簡」、「高質感」的設計要求，無雜亂排版或不協調的配色。需經由使用者確認是否達到類似 Apple 官網的質感。
3.  **動畫流暢度驗收**：
    *   所有轉場與捲動動畫必須保持 60fps 的流暢度，無明顯卡頓。
    *   在行動裝置上動畫需適度簡化或優化，避免過度消耗效能。
4.  **內容正確性**：自我介紹文字需通順無誤，作品集連結需正確指向目標網頁，無無效連結 (Dead Links)。
5.  **作品集導航**：點擊任一作品集連結，應能順利開啟新分頁或跳轉至對應頁面。
6.  **版面適應性**：
    *   在寬螢幕 (Desktop) 下，內容應置中或合理排版，不過度拉伸。
    *   在手機螢幕 (Mobile) 下，內容應自動堆疊，文字大小適中易讀，無水平捲軸出現。
